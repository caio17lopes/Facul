Notação para o diagrama de casos de uso 

    Caso de Uso 
        Um caso de uso pode ser definido, de forma objetiva, como uma   funcionalidade do sistema que será desenvolvido, ou seja, um requisito funcional
        mapeado. Além disso, representa uma ou mais ações que o sistema executará. O elemento que representa o caso de uso é uma elipse contendo no seu interior
        o nome da ação que será executada. Esse nome deve ser formado pela união de um verbo e um substantivo. Na Figura 2, note que existem dois casos de
        uso, o primeiro chamado Realizar pedido, cujo nome é junção do verbo realizar e do substantivo pedido; o segundo, Efetuar pagamento, junção do verbo efetuar
        e do substantivo pagamento  
    
    Ator   
        O ator representa uma entidade externa que possui algum tipo de interação com o sistema, por meio de uma entrada de dados ou uma resposta
        processada. O ator pode ser um usuário desempenhando determinado papel na execução do sistema, um dispositivo de hardware ou até mesmo outro sistema.
        Ou seja, o ator é quem demandará a execução dos casos de uso. O elemento que representa o ator é um boneco, conhecido popularmente por boneco-palito.
        Para identificá-lo, deve-se colocar, logo abaixo da representação, o seu nome (papel assumido pelo ator), com a primeira letra em maiúsculo. A Figura 3 mostra
        alguns exemplos de atores.
    
    Associação 
        O relacionamento entre atores e casos de uso é chamado de associação. Sua função é indicar que determinado ator utiliza ou executa uma ou mais
        funcionalidades (casos de uso) do sistema representado no diagrama de casos de uso. O elemento que representa uma associação é uma linha sólida entre o
        ator e o caso de uso. Como mostra a Figura 4, o ator intitulado Cliente executa os casos de uso Realizar pedido e Realizar pagamento.
        O uso da linha sólida indica que a interação entre o ator e o caso de uso ocorre nos dois sentidos. Caso seja necessário indicar de forma explícita a
        direção em que a interação ocorre (navegabilidade), deve-se utilizar uma linha sólida com uma seta em uma das extremidades
        Outra propriedade que também pode ser descrita no diagrama de caso de uso é a multiplicidade (cardinalidade) existente entre os atores e os casos de
        uso. A multiplicidade tem como objetivo apresentar de forma explícita quantos elementos estão interagindo, em uma associação. A Figura 6 mostra um
        exemplo de multiplicidade. Note que, em cada uma das extremidades da linha, existem indicadores (valores), descritos, que identificam a quantidade de
        elementos envolvidos nessa associação. No caso, a interação presente na

Herança e relacionamentos 
    Herança é o relacionamento dos casos de uso 
    Permite que um caso obetenha definições e funcionalidades de outro caso 
    Os casos especializados herdam as características do caso em geral 

    Herança entre atores especializados herdam todas as características do ator geral
    Casos de uso e relacionamentos 
    representada por uma linha solida com uma seta na extremidade 

    Relacionamento de inclusão entre casos de uso
    Indica a obregatoriedade do caso de incçusão 
    Representada por uma linha tracejadad com uma seta na extremidade, apontando para o caso de inclusão 
    Estereótipo <<include>> 

    Extensão de relacionamento entre casos de uso 
    Será execurado se determinada condição for satisfeita
    Representada por uma linha tracejada com uma seta na extremidade, apontando para o lado oposto do caso estendido 
    Estereótipo <<extend>>

    Especificação do caso de uso 

    Ser objetivo e descrever em detalhes e de forma textual as interaçõe entre o ator e o sistema 
    Existe ua sequência lógica de execução 
    As especificações deve seguir uma estrutura de tópicos para facilitar o entendiemnto 

    A especificação composta pela seguinte estrutura:

    Nome de caso de uso         /   Cadastrar usuario
    Atores                      /   Cliente, restaurante e entregador
    Descrição                   /   Permitir o cadastro de usuários no aplicativo
    Pré-condições               /   Possuir documentos de identificação, dados bancários
    Pós-condições               /   Dados amazenados no banco de dados, permitindo acesso à aplicação
